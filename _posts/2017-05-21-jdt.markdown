Java Day Tokyoに参加してきましたので個人的に刺さった話をまとめておきます。  

### Java Day Tokyo
#### Antifragile Java
TIS株式会社さんの[@kawasimaさん](https://twitter.com/kawasima/)のセッション。  
  
Antifragile（アンチフラジャイル）とは、大きな変化が生じた際に大きな利益を得られる状態のことを指していて、この概念をソフトウェア開発に応用する際に、何を意識してどのように実践していくのかを発表されていました。  

Antifragileなシステム、大きな変化が生じた際に大きな利益を得られるシステムとはどんなシステムだろうと考えたのですが、意図的に障害を発生させることで、システムの欠陥を発見し、改修するというサイクルを回していくことで、システム全体として強くなるシステムのことなのかなと理解してます。  

そのシステムを実現するためには、システムのアーキテクチャから、実装、テスト、リリース、運用などシステム全体の最適化が必須なのかなと思っています。  

セッション中に@kawasimaさんが[Release It!](https://www.amazon.co.jp/Release-%E6%9C%AC%E7%95%AA%E7%94%A8%E3%82%BD%E3%83%95%E3%83%88%E3%82%A6%E3%82%A7%E3%82%A2%E8%A3%BD%E5%93%81%E3%81%AE%E8%A8%AD%E8%A8%88%E3%81%A8%E3%83%87%E3%83%97%E3%83%AD%E3%82%A4%E3%81%AE%E3%81%9F%E3%82%81%E3%81%AB-Michael-T-Nygard/dp/4274067491)を参考に資料を作ったと仰ってたので、即購入しました。  
  
ちょっとずつ読み進めてますが、実際に障害が発生した際の状況と、その原因、そしてどのように解決し、そもそもどうすれば良かったのか、という点が記述されており、大変参考になる書籍です。  

話は変わって本セッションがアプリケーション開発者側からの視点だとすれば、運用側からの視点で変化に強いシステムを目指していく手法を語っているのが[@y_uuk1さん](https://twitter.com/y_uuk1)で、この方の[高度に発達したシステムの異常は神の怒りと見分けがつかない](http://blog.yuuk.io/entry/ipsjone2017)からは非常に影響を受けており、自分の現場でもシステムメトリクスの収集から、ログ解析基盤の導入などができないかと考えています。  
  
個人的な命題として、幸せなシステム開発を目指すというざっくりな目標をもっていたのですが、今回このセッションに参加することでより具体的なイメージをもつことができました。それがAntifragileなシステム開発という言葉で表現されることで、そのようなシステムを目指すためのヒントも得られることができました。  
  
ヒントをヒントで終わらせないためにも、今後も継続的にIN/OUTを繰り返していけたらと思います。
